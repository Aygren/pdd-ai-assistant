🚘 PDD Assistant: Мультиплатформенная ИИ-экосистема Автоюриста

Интеллектуальный ИИ-помощник для анализа обстоятельств ДТП и юридической консультации на основе актуальных Правил дорожного движения РФ. Проект включает в себя веб-интерфейс и умного Telegram-бота с поддержкой голосового ввода.

🔥 Ключевая особенность (Архитектура)

В отличие от стандартных RAG-систем, проект реализует продвинутый паттерн Information Gathering Agent (Квалифицирующий агент):

Сбор фактов: Модель анализирует дорожную ситуацию по чек-листу ГИБДД через структурированный JSON-анализ.

Дорожная логика: ИИ понимает иерархию правил (например, регулируемый перекресток и светофоры отменяют знаки приоритета), а также распознает специфичные типы аварий (например, попутные столкновения и наезды сзади), автоматически адаптируя диалог.

Уточняющие вопросы: Если данных для вердикта мало, бот сам задает точечные вопросы водителю в чате, удерживая контекст сессии.

Гибридный RAG: Финальный юридический разбор формируется только после сбора 100% необходимых фактов на основе семантического поиска по базе знаний.

🌐 Архитектура экосистемы

Проект развернут как распределенная система, где один бэкенд обслуживает два независимых интерфейса:

Фронтенд-сайт: Развернут на Streamlit Cloud для пользователей, предпочитающих веб-интерфейс.

Telegram-бот: Развернут на Render.com в виде отказоустойчивого веб-сервиса с круглосуточным планировщиком (Cron-Job.org), защищающим сервер от «засыпания».

🛠 Стек технологий

Язык: Python 3.12

Фреймворк оркестрации: LangChain (цепочки с сохранением контекста в памяти InMemoryChatMessageHistory)

LLM (Мозг системы): Mistral AI (mistral-small-latest) с поддержкой строгого JSON-mode

STT (Распознавание речи): Groq API (whisper-large-v3) — сверхбыстрая бесплатная транскрибация голосовых сообщений для Telegram-бота.

База знаний (Vector Store): Supabase (pgvector)

Интерфейсы: Streamlit (веб) и pyTelegramBotAPI (мессенджер).

🚀 Инструкция по локальному развертыванию и настройке

Выполните последовательно следующие команды в терминале:

# 1. Клонирование репозитория и переход в папку
git clone <url_твоего_репозитория>
cd pdd-ai-assistant

# 2. Создание виртуального окружения
python -m venv .venv

# 3. Активация окружения
# Для Linux/macOS:
source .venv/bin/activate
# Для Windows:
# .venv\Scripts\activate

# 4. Установка необходимых библиотек (включая pyTelegramBotAPI и OpenAI-клиент)
pip install -r requirements.txt


5. Настройка окружения (.env)

Создайте файл .env в корне проекта и добавьте ваши ключи:

MISTRAL_API_KEY=your_mistral_api_key_here
SUPABASE_URL=your_supabase_project_url_here
SUPABASE_KEY=your_supabase_anon_public_key_here
TG_BOT_TOKEN=your_telegram_bot_token_from_botfather
GROQ_API_KEY=your_groq_api_key_for_whisper


6. Запуск компонентов локально

Запуск веб-сайта:

  streamlit run app.py


Запуск Telegram-бота:

  python tg_bot.py


☁️ Облачный деплой (Production)

1. Веб-интерфейс (Streamlit Cloud)

Проект подключен напрямую к GitHub и автоматически обновляется при пуше в ветку main. Переменные окружения задаются в панели управления Streamlit в разделе Advanced Settings.

2. Telegram-бот (Render.com)

Бот развернут как Web Service на бесплатном тарифе со специальной командой запуска, поднимающей пустой HTTP-сервер для прохождения проверок платформы:

Build Command: pip install -r requirements.txt

Start Command: python tg_bot.py & python -m http.server $PORT

3. Оптимизация доступности (Cron-Job.org)

Для предотвращения «засыпания» бесплатного контейнера на Render настроен внешний крон-сервис, который каждые 15 минут отправляет GET-запрос на URL-адрес веб-службы, обеспечивая мгновенный отклик Telegram-бота в любое время суток.
